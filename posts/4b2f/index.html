网络流

发表于2023-03-19|更新于2023-03-23|算法图论
|阅读量:

参考：OI Wiki|王树深

名词：最大流，阻塞流

最大流一定是阻塞流，但是阻塞流不一定是最大流

m是边，n是节点，f为最大流的大小

Ford-Fulkerson Algorithm（算法复杂度，依赖于最大流的大小）

key concept:residual graph

  1. 构造residual graph
  2. 寻找简单路径
  3. 取路径上residual最小的管道流量作为路径上能通过的最大的流量，然后路径上每个管道都减去这个
  4. 算法依次进行，直到找不到从到的简单路径

Edmonds-karp Algorithm（算法复杂度，不依赖于最大流的大小）

key concept:residual graph

  1. 构造residual graph
  2. 寻找简单路径
  3. 取路径上residual最小的管道流量作为路径上能通过的最大的流量，然后路径上每个管道都减去这个
  4. 然后按照给这个路径上的管道都建一条反向的边，其流量为
  5. 算法依次进行，直到找不到从到的简单路径

Dinic‘s Algorithm（算法复杂度为）

key concept:block flow、level graph

  1. 构造residual graph
  2. iteration
    1. 根据residual graph 构造level graph
    2. 在level graph上找阻塞流
    3. 用找到的阻塞流来更新residual graph，并在residual graph中建立反向边，其流量大小与阻塞流大小相同
