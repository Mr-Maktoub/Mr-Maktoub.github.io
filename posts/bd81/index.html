    1. 选取合适的验证方法。比如AUC，mAP(mean average precision)，micro/macro F-beta score 等，根据任务和数据分布选择最适合的验证方法。一般来说，ACC(Precision)是较为不可靠的验证指标，尤其是正负样本不均衡的情况下(比如正负样本9:1,全预测1也有90%分)。
    2. 相信本地交叉验证结果而不是线上LB。因为线上测试数据量更小，而交叉验证数据量却是整个训练数据集。
    3. 相信自己。一些降低模型预测方差的方法，如TTA（test-time augmentation），Blending等集成方法，能降低shake风险（防止public LB分很高而private LB分很低）。一些自己的经验相信有效的方法，不要因为不稳定的测试集而轻易放弃。
    4. 对于CV任务，优秀的特征提取更为重要。使用metric learning等方法可以获取更好的特征提取，这样的模型通常会更鲁棒更值得相信。（目前我只提交了baseline，还尚未尝试这点）。

随机种子

这段代码主要是为了设置随机种子，以确保在使用NumPy和PyTorch进行随机操作时得到可重复的结果。然而，有几个问题需要注意：

  1. 使用随机种子可以确保在相同的种子下运行时获得相同的随机数序列。但是，这并不意味着代码的其他部分不会对结果产生影响。如果您在代码中使用了其他具有随机性的操作（例如随机初始化模型权重），那么即使使用相同的种子，也无法完全复现结果。
  2. 对于第一行代码 os.environ['PYTHONHASHSEED'] = str(seed)，请注意这是为了设置Python的哈希种子，与随机数生成无关。如果您不打算使用Python的哈希函数，可以忽略这行代码。
  3. 在使用torch.backends.cudnn.deterministic = True时，需要注意它会导致一些运行时性能下降。这是因为它会禁用某些针对提高运行速度的优化。如果您对结果的可重复性要求不高，可以将其设置为False，以获得更好的性能。

感受野

在深度学习中，感受野（receptive field）指的是神经网络中一个特定神经元对输入数据的感知范围。换句话说，它表示了神经元能够”看到”或”感受到”输入数据的区域大小。

感受野的大小取决于网络的结构和层级。在卷积神经网络（Convolutional Neural Network，CNN）中，每个卷积层的神经元只与前一层中局部区域的神经元连接。通过堆叠多个卷积层，神经元的感受野逐渐扩大，使网络能够对更大范围的输入进行学习和理解。

在CNN中，感受野的大小通常通过滤波器的尺寸（或卷积核的大小）和步幅（stride）来确定。滤波器的尺寸定义了每个卷积层的局部感受野大小，而步幅定义了滤波器在输入上滑动的步长。通过调整这两个参数，可以控制感受野的大小和网络对输入的感知程度。

在深度网络中，较低层的神经元具有较小的感受野，而较高层的神经元具有更大的感受野。这是因为较低层的神经元主要负责提取局部特征，而较高层的神经元则负责整合更大范围的信息，从而理解更高级的语义和语境。

总结起来，感受野是指神经网络中神经元对输入数据的感知范围，它的大小取决于网络的结构、卷积层的尺寸和步幅。感受野的逐层扩大是深度网络能够对输入数据进行更全局、更抽象理解的重要因素之一。

8-bit 的灰度图像

8-bit 的灰度图像是一种灰度图像，其中每个像素的像素值用 8 个比特（或 1 个字节）表示。在这种图像中，每个像素的像素值范围从 0 到 255，共有 256 个不同的可能值。

8-bit 的灰度图像通常用于表示黑白图像，其中较小的像素值表示较暗的像素，较大的像素值表示较亮的像素。像素值越低，像素越接近黑色；像素值越高，像素越接近白色。

在这样的图像中，每个像素的像素值占用 8 个比特，因此可以表示的不同灰度级别为 2^8 = 256 级。这使得 8-bit 灰度图像在灰度表示方面具有较高的分辨率，能够准确地表示较细微的灰度变化。

在计算机视觉和图像处理领域，8-bit 灰度图像常用于各种任务，包括图像预处理、特征提取、图像分析和计算机视觉算法的训练和评估等。

model and mode

在计算机科学领域中，”model” 和 “mode” 是两个不同的概念。

  1. Model (模型)：在机器学习和深度学习中，”model” 指的是经过训练的算法或神经网络模型。这些模型通过从数据中学习来建立预测或分类任务的规律。模型可以根据输入数据生成相应的输出，用于解决各种问题，例如图像分类、语言生成、预测等。训练模型通常涉及数据预处理、模型构建、优化算法和参数调整等步骤。
  2. Mode (模式)：”mode” 通常表示某种状态或操作模式。它可以应用于不同的计算领域，如软件开发、系统设计和用户界面等。在软件开发中，”mode” 可以指示程序运行的不同状态或配置。例如，一个文本编辑器可以有编辑模式和阅读模式，用户可以切换模式以执行不同的操作。在系统设计中，”mode” 可以指示系统的工作状态，如调试模式、生产模式等。在用户界面设计中，”mode” 可以表示用户界面的交互模式，例如命令模式和图形模式。

总结：在机器学习和深度学习中，”model” 表示训练好的算法或神经网络模型；而在计算领域中的其他方面，”mode” 表示某种状态、配置或操作模式。这两个概念在不同的上下文中具有不同的含义和用途。
